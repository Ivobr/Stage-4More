# Backlog & Planning

## Backlog uitleg

De product backlog is opgedeeld in 2 verschillende tabellen voor het gemak Orginele tabel en controller tabel te noemen.
De orginele tabel is gemaakt aan het begin van mijn stage periode en hierin zijn alle orginele requirments te vinden.
Nadat ik deze backlog bijna had afgerond is de controller tabel erbij gekomen.  
Zoals te zien is is de orginele backlog
niet helemaal afgerond maar ben ik wel begonnen met de controller backlog.  
Dit is omdat er voor user story 3 (Grijper ontwerpen) een 3D printer nodig is welke tot begin deze week (stage week
1.10) defect was hierdoor heb ik deze user story nog niet kunnen afronden. Ook user story 6 is niet afgewerkt dit omdat
de taken welke uitgevoerd moeten worden aan de productie lijn uitgevoerd zouden moeten worden. Aangezien het hortensia
seizoen nu voorbij is kan dit niet meer gedaan worden.

User story 4 staat als afgerond gemarkeerd zonder alle taken te hebben voltooid. Dit is omdat de acceptatiecriteria wel
is bereikt maar omdat de grijper nog niet optimaal is zal deze test nog vaker worden uitgevoerd met een nieuw ontworpen
grijper.

Voor functionaliteit 3 is er nog geen goede benaming gevonden voor de user story.
